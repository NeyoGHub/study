## 🔬 关键技术要点

### 1. 财务数据双数据源

东方财富(em)和同花顺(ths)各有优劣：

```python
# 东方财富：财务摘要指标全面，但三张报表接口有bug
df_abstract = ak.stock_financial_abstract(symbol="000001")

# 同花顺：三张报表稳定可用
df_profit = ak.stock_financial_benefit_ths(symbol="000001")
df_debt = ak.stock_financial_debt_ths(symbol="000001")
df_cash = ak.stock_financial_cash_ths(symbol="000001")
```

### 2. 财务分析指标的86个维度

`stock_financial_analysis_indicator` 返回86个财务指标，关键指标：

| 指标 | 说明 |
|------|------|
| 摊薄每股收益(元) | 基本EPS |
| 加权每股收益(元) | 加权EPS |
| 每股净资产_调整前(元) | 每股账面价值 |
| 每股经营性现金流(元) | 经营现金流/股本 |
| 净资产收益率(%) | ROE |
| 销售净利率(%) | 净利率 |
| 资产负债率(%) | 杠杆率 |

### 3. financial_abstract 的数据结构

返回80行指标 × 124个时间列，行名在"指标"列：

```python
df = ak.stock_financial_abstract(symbol="000001")
roe = df[df['指标'] == '净资产收益率']
print(roe.iloc[0, 3:10])  # 最近几期的ROE
```

## 💡 重要发现

### 发现1: 东方财富三张报表接口有bug

`stock_profit_sheet_by_report_em`、`stock_balance_sheet_by_report_em`、`stock_cash_flow_sheet_by_report_em` 三个接口内部有 `NoneType` 错误，属于库本身的问题。建议使用同花顺(ths)版本替代。

### 发现2: 同花顺财务数据格式复杂

同花顺的财务数据字段名为中文长句，如 `*归属于母公司所有者的净利润`，解析时需要注意。报表核心指标行提供快速摘要。

### 发现3: 深市数据比沪市丰富

`stock_info_sz_name_code` 包含板块分类、行业、总股本、流通股本，而 `stock_info_sh_name_code` 信息较少。
