## 🔬 关键技术要点

### 1. 数据源特点

| 数据源 | 优势 | 劣势 |
|--------|------|------|
| **东方财富** | 基金数据最全面 | ETF历史数据当前不可用 |
| **同花顺** | ETF行情稳定 | 仅ETF数据 |
| **新浪财经** | ETF/LOF行情稳定 | 无基金净值数据 |

### 2. 基金代码格式

基金代码均为 **6位数字**（如 `000001`），不需要市场前缀（区别于股票代码的 sh/sz 格式）。

### 3. ETF历史数据获取

ETF（交易型开放式指数基金）在交易所上市交易，历史数据可通过以下方式获取：

```python
# 方式1：fund_etf_hist_em（东方财富）
df = ak.fund_etf_hist_em(symbol="510050", period="daily",
                          start_date="20250101", end_date="20260507")

# 方式2：stock_zh_a_daily（新浪，ETF可像股票一样交易）
df = ak.stock_zh_a_daily(symbol="sh510050", adjust="qfq")
```

### 4. 基金排行维度

`fund_open_fund_rank_em` 提供丰富的收益率维度：
- **短期**: 近1周、近1月、近3月
- **中期**: 近6月、近1年
- **长期**: 近2年、近3年
- **特殊**: 今年来、成立来、自定义

### 5. 持仓数据结构

基金持仓数据按 **季度** 更新，包含：
- 前十大重仓股及占比
- 累计买入/卖出明细
- 行业配置分布

## 💡 重要发现

### 发现1: 基金数据源与股票不同

基金数据主要来自 **东方财富**（而股票核心数据源是新浪财经）。东方财富的基金数据接口非常稳定，仅ETF历史数据偶有连接问题。

### 发现2: ETF可以当股票查

ETF在交易所上市交易，可以用 `stock_zh_a_daily`（股票日线接口）查询历史数据，作为 `fund_etf_hist_em` 的回退方案。

### 发现3: 基金代码统一为6位数字

与股票代码需要 `sh`/`sz` 前缀不同，基金代码统一使用6位纯数字（如 `000001`），使用上更简单。

### 发现4: 基金接口数量庞大

AkShare 有 **102个** fund 相关接口，覆盖几乎所有基金数据类型，但部分接口存在稳定性问题（`fund_scale_open_sina` 有库bug，`fund_etf_hist_em` 偶发连接失败）。
