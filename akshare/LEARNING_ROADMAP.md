## 学习方法论

### 每个阶段的标准流程

1. **数据源调研**（2小时）
   - 查阅官方文档
   - 了解数据源和接口
   - 识别主要接口

2. **接口测试**（2-3小时）
   - 编写测试脚本
   - 测试所有主要接口
   - 记录数据字段和格式

3. **数据分析**（2小时）
   - 分析数据质量
   - 评估数据完整性
   - 测试数据稳定性

4. **示例开发**（2-3小时）
   - 开发实用示例
   - 添加数据可视化
   - 编写使用文档

5. **文档整理**（1小时）
   - 编写研究报告
   - 更新项目总结
   - 整理代码和脚本

### 标准化输出

每个阶段完成后，必须产生以下输出：

#### 1. 测试脚本（scripts/）
```
scripts/
├── test_[category].py          # 数据接口测试脚本
├── example_[number]_[category]_data.py  # 实用示例
└── output/                     # 测试输出数据
```

#### 2. 研究文档（docs/）
```
docs/
└── [category]_study.md         # 数据类别研究文档
```

#### 3. 项目更新
- 更新 `PROJECT_SUMMARY.md`
- 更新 `README.md`
- 更新学习进度
