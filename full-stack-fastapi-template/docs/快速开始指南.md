# FastAPI全栈模板快速开始指南

> 快速上手 FastAPI 全栈开发环境

## 📋 前置要求

在开始之前，请确保您的系统已安装以下软件：

- **Docker**: 20.10+
- **Docker Compose**: 最新版本
- **Python**: 3.9+ (可选，用于运行脚本)
- **Node.js**: 18+ (可选，本地开发需要)

### 检查安装

```bash
# 检查Docker
docker --version

# 检查Docker Compose
docker compose version

# 检查Python (可选)
python3 --version

# 检查Node.js (可选)
node --version
```

## 🚀 快速启动

### 方法一：自动化脚本（推荐）

```bash
# 1. 进入项目目录
cd /home/neyo/workspace/code/study/full-stack-fastapi-template

# 2. 运行环境搭建脚本
python3 scripts/01_basic_setup.py

# 3. 等待脚本完成（首次可能需要10-20分钟）
```

脚本会自动：
- 检查前置条件
- 设置环境变量
- 构建并启动Docker服务
- 检查服务状态
- 生成环境报告

### 方法二：手动启动

```bash
# 1. 进入源码目录
cd /home/neyo/workspace/code/study/full-stack-fastapi-template/src/full-stack-fastapi-template

# 2. 复制环境变量文件
cp .env.example .env

# 3. 启动所有服务
docker compose up --build -d

# 4. 查看服务状态
docker compose ps

# 5. 查看日志
docker compose logs -f
```

## 📝 环境变量配置

### 主要环境变量

编辑 `.env` 文件配置以下变量：

```bash
# 项目名称
PROJECT_NAME=backend

# 后端配置
BACKEND_CORS_ORIGINS=http://localhost,http://localhost:4200,http://localhost:3000
SECRET_KEY=changethisinproduction

# 数据库配置
POSTGRES_USER=app
POSTGRES_PASSWORD=changethisinproduction
POSTGRES_SERVER=db
POSTGRES_DB=app

# 邮件配置（可选）
SMTP_TLS=True
SMTP_PORT=587
SMTP_HOST=smtp.gmail.com
SMTP_USER=your-email@example.com
SMTP_PASSWORD=your-email-password
EMAILS_FROM_EMAIL=noreply@example.com
```

### 安全提示

⚠️ **生产环境必须修改以下变量**：
- `SECRET_KEY`: 使用强随机密钥
- `POSTGRES_PASSWORD`: 使用强密码
- `SMTP_PASSWORD`: 使用邮件账户密码

## 🛠️ 常用命令

### Docker Compose 操作

```bash
# 启动服务
docker compose up -d

# 停止服务
docker compose down

# 重启服务
docker compose restart

# 查看日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f backend
docker compose logs -f frontend
docker compose logs -f db

# 重新构建并启动
docker compose up --build -d

# 进入容器
docker compose exec backend bash
docker compose exec frontend sh
docker compose exec db psql -U app
```

### 数据库操作

```bash
# 连接到数据库
docker compose exec db psql -U app

# 在psql中执行SQL
\dt                    # 列出所有表
\d users              # 查看users表结构
SELECT * FROM users;   # 查询users表数据
\q                    # 退出psql

# 数据库迁移
docker compose exec backend alembic upgrade head
```

### 后端开发

```bash
# 进入后端容器
docker compose exec backend bash

# 运行测试
pytest

# 运行特定测试
pytest tests/test_api.py

# 创建新的迁移
alembic revision --autogenerate -m "description"

# 应用迁移
alembic upgrade head
```

### 前端开发

```bash
# 进入前端容器
docker compose exec frontend sh

# 安装新依赖
npm install package-name

# 运行开发服务器（容器内已运行）
# npm run dev

# 构建生产版本
npm run build
```

## 🔧 故障排除

### 1. 端口已被占用

**问题**: `Error starting userland proxy: listen tcp 0.0.0.0:80: bind: address already in use`

**解决方法**:
```bash
# 检查端口占用
sudo lsof -i :80
sudo lsof -i :8000
sudo lsof -i :5432

# 停止占用端口的服务或修改端口映射
# 编辑 docker-compose.yml 或 compose.yml 文件
# 修改 ports 配置
```

### 2. Docker 镜像构建失败

**问题**: 构建过程中出现错误

**解决方法**:
```bash
# 清理Docker缓存
docker builder prune -a

# 重新构建
docker compose build --no-cache

# 检查网络连接
ping google.com
```

### 3. 数据库连接失败

**问题**: 后端无法连接到数据库

**解决方法**:
```bash
# 检查数据库服务状态
docker compose ps

# 查看数据库日志
docker compose logs db

# 重启数据库
docker compose restart db

# 检查环境变量配置
cat .env | grep POSTGRES
```

### 4. 容器无法启动

**问题**: 容器启动后立即退出

**解决方法**:
```bash
# 查看容器日志
docker compose logs backend
docker compose logs frontend

# 检查容器状态
docker compose ps -a

# 进入容器调试
docker compose exec backend bash
```

### 5. 内存不足

**问题**: 系统内存不足，容器启动失败

**解决方法**:
```bash
# 减少同时运行的服务数量
# 编辑 docker-compose.yml
# 移除或注释不需要的服务

# 增加系统交换空间
# （需要root权限）
```
